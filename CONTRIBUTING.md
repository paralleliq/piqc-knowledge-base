# Contributing

Thank you for your interest in contributing!  
This project is part of the ParallelIQ Knowledge Base and aims to provide a neutral, framework-agnostic resource for ML/LLM deployment quality.

Your contributions help improve operational practices for the entire ecosystem.

---

## How to Contribute

### 1. Open an Issue
Before submitting a PR, please open an Issue describing:

- The problem or improvement you propose  
- Why it is valuable  
- Any examples or references  

This helps maintain clarity and prevents duplicate work.

### 2. Submit a Pull Request
Once the issue is acknowledged:

1. Fork the repository  
2. Create a new branch: `feature/<short-name>`  
3. Make your changes  
4. Submit a PR referencing the Issue number  

Please keep contributions:

- Concise  
- Actionable  
- Framework-neutral  
- Vendor-neutral  

### 3. Checklist Contribution Guidelines
When adding or modifying checklist items:

- Avoid product-specific recommendations  
- Use clear wording (one idea per item)  
- Prefer actionable checks ("X is configured")  
- Avoid subjective statements  
- Validate that items apply broadly across ML systems  

### 4. Style Guidelines

- Use GitHub checkbox syntax (`- [ ]`)  
- Use section headers (`##`) consistently  
- Keep descriptions short and operational  
- Use American English spelling  

### 5. Code of Conduct
By participating in this project, you agree to uphold respectful and constructive discussion.  
Disrespectful or non-collaborative behavior will not be accepted.
